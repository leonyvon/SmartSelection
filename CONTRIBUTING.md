<a name="english"></a>
## English

Thank you for your interest in contributing to SmartSelection! This document provides guidelines for contributing.

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/SmartSelection.git`
3. Install dependencies: `pnpm install`
4. Start development server: `pnpm dev`

### Development Workflow

1. Create a new branch for your feature/fix: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test your changes thoroughly
4. Commit with clear messages
5. Push to your fork and create a Pull Request

### Code Style

- Use TypeScript for all new code
- Follow existing code patterns
- Keep components small and focused
- Add appropriate type definitions

### Pull Request Guidelines

- Provide a clear description of changes
- Reference any related issues
- Ensure code builds successfully
- Test on both Windows and macOS if possible

### Reporting Issues

- Use GitHub Issues
- Provide clear reproduction steps
- Include system information (OS, Node version, etc.)
- Include screenshots if applicable

<a name="中文"></a>
## 中文

感谢你对 SmartSelection 的贡献兴趣！本文档提供了贡献指南。

### 开始之前

1. Fork 本仓库
2. 克隆你的 Fork: `git clone https://github.com/YOUR_USERNAME/SmartSelection.git`
3. 安装依赖: `pnpm install`
4. 启动开发服务器: `pnpm dev`

### 开发流程

1. 为你的功能/修复创建新分支: `git checkout -b feature/your-feature-name`
2. 进行修改
3. 充分测试你的修改
4. 使用清晰的提交信息提交
5. 推送到你的 Fork 并创建 Pull Request

### 代码风格

- 所有新代码使用 TypeScript
- 遵循现有代码模式
- 保持组件小而专注
- 添加适当的类型定义

### Pull Request 指南

- 提供清晰的修改描述
- 引用相关的 Issue
- 确保代码能够成功构建
- 如可能，在 Windows 和 macOS 上测试

### 报告问题

- 使用 GitHub Issues
- 提供清晰的复现步骤
- 包含系统信息（操作系统、Node 版本等）
- 如适用，包含截图
